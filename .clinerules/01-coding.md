1. Use shadcn-svelte components (install if needed)
2. Backend-first workflow:
   - Define logic & types in backend (Pydantic models)
   - Run `npm run models` to sync types to frontend
   - Use generated types with api.ts client (openapi-fetch)
3. Never edit lib/types/api.ts manually - it's autogenerated
4. New components go in lib/components/ (/ui is for shadcn only)
5. Always handle loading and error states, add skeletons when logical
